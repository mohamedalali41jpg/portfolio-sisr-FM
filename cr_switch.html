Switch Cisco & SSH

Ce projet portait sur la configuration initiale d'un commutateur réseau (Switch Cisco) via l'interface en ligne de commande (CLI). L'accent a été mis sur les bonnes pratiques de sécurité : création de VLANs, protection par mots de passe chiffrés, et sécurisation des accès distants (VTY) via le protocole SSH.

2. Configuration Initiale et Sécurisation Base

Les premières étapes permettent d'identifier l'équipement et de le protéger physiquement et logiquement :

  • Attribution d'un nom (hostname SW-CORE-01) pour l'identifier sur le réseau.
  • Protection du mode privilégié avec la commande enable secret [mot_de_passe], qui utilise un hachage fort (MD5) contrairement à enable password.
  • Chiffrement de tous les mots de passe stockés en clair dans le fichier de configuration avec service password-encryption.

3. Déploiement de SSH (Désactivation de Telnet)

Pour contrer les failles liées à Telnet (analyse de paquets démontrée dans le projet Wireshark), le SSH a été implémenté :

  • Configuration du domaine IP : ip domain-name entreprise.local.
  • Génération des clés cryptographiques RSA de 1024 bits : crypto key generate rsa.
  • Création d'un compte local pour l'authentification : username admin privilege 15 secret [mdp].
  • Configuration des lignes VTY pour n'accepter que le protocole SSH (transport input ssh) et obliger l'authentification locale (login local).
