Configuration DHCP

L'objectif de ce projet réseau était de déployer le rôle DHCP (Dynamic Host Configuration Protocol) sur une instance Windows Server 2022. Cela permet d'automatiser et de centraliser l'attribution des adresses IP dans un réseau d'entreprise, tout en évitant les conflits d'adresses.

1. Processus DORA et Topologie

sequenceDiagram participant Client as Poste Client (Sans IP) participant DHCP as Windows Server 2022 (DHCP) Client->>DHCP: 1. DISCOVER (Broadcast) DHCP-->>Client: 2. OFFER (Proposition IP) Client->>DHCP: 3. REQUEST (Demande officielle) DHCP-->>Client: 4. ACKNOWLEDGE (Validation et Bail)

2. Mise en place de l'Étendue (Scope)

L'étendue définit la plage d'adresses IP que le serveur a le droit de distribuer :

  • Création d'une nouvelle étendue IPv4 (ex: 192.168.10.10 à 192.168.10.100).
  • Définition du masque de sous-réseau (ex: /24 soit 255.255.255.0).
  • Configuration des options d'étendue vitales : Routeur par défaut (Passerelle) et Serveurs DNS.

3. Exclusions et Réservations

Pour des raisons d'administration, certains équipements (imprimantes, serveurs annexes) ne doivent pas changer d'IP :

  • Exclusions : Retrait manuel des IP de 192.168.10.10 à 192.168.10.20 pour qu'elles ne soient jamais distribuées (réservées aux administrateurs réseau).
  • Réservations : Liaison statique de l'adresse MAC d'une imprimante réseau avec une adresse IP spécifique dans le pool DHCP, garantissant que cette imprimante aura toujours la même IP tout en gardant une gestion centralisée.
